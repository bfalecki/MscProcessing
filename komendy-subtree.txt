Zasada nr 1 (najważniejsza)

Commitujesz ZAWSZE w repo-A
processing/ traktujesz jak zwykły folder
repo-B jest synchronizowane jawnie przez subtree


Synchronizacja do repo-B:

git subtree push --prefix=processing https://github.com/bfalecki/MscProcessing.git main

To wycina tylko processing/
aktualizuje repo-B
bez duplikacji plikow



PULL – jeśli repo-B się zmieniło
(np. ktoś inny coś tam zcommitował)

git subtree pull --prefix=processing https://github.com/bfalecki/MscProcessing.git main --squash



Dodaj remote (jednorazowo)
git remote add repoB https://github.com/bfalecki/MscProcessing.git

Alias push
git config alias.push-processing "subtree push --prefix=processing repoB main"

git config alias.pull-processing "subtree pull --prefix=processing repoB main --squash"

Wtedy tylko 
git push-processing
git pull-processing
